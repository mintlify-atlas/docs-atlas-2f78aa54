## IBiddingParticipantService

Manages bidding participants in public procurement processes.

**Namespace:** `services`  
**Implementation:** `BiddingParticipantService`

### Dependencies

<ParamField path="_context" type="ApplicationDbContext">
  Entity Framework database context
</ParamField>

<ParamField path="_mapper" type="IMapper">
  AutoMapper for DTO conversions
</ParamField>

<ParamField path="_uploadedFileIIS" type="IUploadedFileIIS">
  File upload service for IIS
</ParamField>

<ParamField path="_emailSendGrid" type="IEmailSendGrid">
  SendGrid email service for notifications
</ParamField>

### Methods

#### GetAll

Retrieves all bidding participants with related Master data.

```csharp
Task<IEnumerable<BiddingParticipant>> GetAll()
```

<ResponseField name="return" type="Task<IEnumerable<BiddingParticipant>>">
  Collection of all participants with included Master relationship
</ResponseField>

**Implementation:**
```csharp
return await _context.BiddingParticipants
    .Include(x => x.Master)
    .ToListAsync();
```

#### GetById

Retrieves a participant by ID.

<ParamField path="id" type="int?" required>
  Participant identifier
</ParamField>

```csharp
Task<BiddingParticipant> GetById(int? id)
```

<ResponseField name="return" type="Task<BiddingParticipant>">
  Participant entity with Master navigation property loaded
</ResponseField>

#### GetByRef

Retrieves a participant by their unique reference GUID.

<ParamField path="reference" type="string" required>
  GUID reference code (converted internally)
</ParamField>

```csharp
Task<BiddingParticipant> GetByRef(string reference)
```

<ResponseField name="return" type="Task<BiddingParticipant>">
  Participant matching the reference code
</ResponseField>

#### Create

Registers a new bidding participant with proposal file and email notifications.

<ParamField body="model" type="BiddingParticipantCreateDTO" required>
  Participant data
</ParamField>

```csharp
Task<BiddingParticipantDTO> Create(BiddingParticipantCreateDTO model)
```

<ResponseField name="process" type="object">
  <Expandable title="creation-process">
    1. Generate unique GUID reference
    2. Upload proposal PDF to `filesBiddingParticipant` folder
    3. Create participant record in database
    4. Generate HTML email from template
    5. Send confirmation emails to:
       - Participant email
       - asesoria@espsantafedeantioquia.co
       - gerencia@espsantafedeantioquia.co
  </Expandable>
</ResponseField>

**Example:**
```csharp
var participant = await _biddingParticipantService.Create(new BiddingParticipantCreateDTO
{
    NaturalPerson = true,
    Name = "Juan Pérez",
    IdentificationOrNit = "1234567890",
    Phone = "6045551234",
    Cellular = "3001234567",
    Address = "Calle 10 #20-30",
    City = "Santa Fe de Antioquia",
    Email = "juan@example.com",
    Proposals = proposalPdfFile,
    MasterId = 5
});
```

#### DuplicaIdentificationOrNit

Validates that an identification/NIT is unique within a specific announcement.

<ParamField path="identificationOrNit" type="string" required>
  ID or tax number to check
</ParamField>

<ParamField path="masterId" type="int" required>
  Announcement/convocatoria ID
</ParamField>

```csharp
Boolean DuplicaIdentificationOrNit(string identificationOrNit, int masterId)
```

<ResponseField name="return" type="Boolean">
  `true` if duplicate exists, `false` if unique
</ResponseField>

#### DeleteConfirmed

Removes a participant and their uploaded proposal file.

<ParamField path="id" type="int" required>
  Participant ID to delete
</ParamField>

```csharp
Task DeleteConfirmed(int id)
```


#### GetById (string)

Retrieves a blog post by URL slug.

<ParamField path="urBlog" type="string" required>
  URL-friendly blog identifier
</ParamField>

```csharp
Task<Master> GetById(string urBlog)
```

#### Create

Creates a new blog post with cover image.

<ParamField body="model" type="BlogCreateDto" required>
  Blog post data
</ParamField>

```csharp
Task<BlogDto> Create(BlogCreateDto model)
```

<ResponseField name="process" type="object">
  <Expandable title="creation-steps">
    1. Generate URL slug from blog name
    2. Upload cover image to `blog` folder
    3. Create Master record with Blog = true
    4. Set creation timestamp
    5. Mark as active (Statud = true)
  </Expandable>
</ResponseField>

**Example:**
```csharp
var blog = await _blogService.Create(new BlogCreateDto
{
    NameBlog = "Nueva Iniciativa 2026",
    Description = "<p>Contenido HTML del artículo...</p>",
    Author = "Comunicaciones ESP",
    CoverPage = imageFile
});
// Creates URL: /noticias/nueva-iniciativa-2026
```

  Blog post ID

<ParamField body="model" type="BlogEditDto" required>

</ParamField>

```csharp

```

<ResponseField name="behavior" type="string">
  If new cover image provided, replaces existing. Otherwise, retains current image.
</ResponseField>


#### DeleteConfirmed

<ParamField path="id" type="int" required>

</ParamField>

```csharp

```

## IDocumentService

**Namespace:** `services`  
**Implementation:** `DocumentService`

<ParamField path="_urlName" type="string" required>
  URL-friendly document name
</ParamField>

```csharp

```

#### Create

Creates a document with multiple file attachments using database transaction.

<ParamField body="model" type="DocumentCreateDTO" required>
  Document metadata and files
</ParamField>

```csharp
Task Create(DocumentCreateDTO model)
```

<ResponseField name="transaction" type="object">
  <Expandable title="transaction-flow">
    1. Begin database transaction
    2. Create Document record
    3. Generate URL slug
    4. Associate with Master if MasterId provided
    5. Loop through uploaded files:
       - Format filename to URL-friendly slug
       - Upload file to `filesDocuments` folder
       - Create FileDocument record
    6. Commit transaction
    7. Rollback on any exception
  </Expandable>
</ResponseField>

**Example:**
```csharp
await _documentService.Create(new DocumentCreateDTO
{
    Name = "Plan de Desarrollo 2026",
    Description = "Documento oficial del plan...",
    MasterId = 10,
    RouteFile = new List<IFormFile> { file1, file2, file3 }
});
```

#### FilesDocuments

Retrieves all file documents.

```csharp
Task<IEnumerable<FileDocument>> FilesDocuments()
```

#### DeleteConfirmed

Deletes a document and all associated files using transaction.

<ParamField path="_id" type="int" required>
  Document ID
</ParamField>

```csharp
Task DeleteConfirmed(int _id)
```

<ResponseField name="process" type="object">
  <Expandable title="deletion-steps">
    1. Begin transaction
    2. Load document with FileDocument collection
    3. Loop through files:
       - Delete physical file from server
       - Remove FileDocument record
    4. Remove Document record
    5. Commit transaction
  </Expandable>
</ResponseField>

#### DocumentExists

Validates document existence.

```csharp
bool DocumentExists(int id)
```

#### DuplicaName

Checks for duplicate document names.

```csharp
bool DuplicaName(string _stringName)
```

## Common Service Utilities

### IFormatStringUrl

Converts text to URL-friendly slugs.

**Example transformations:**
- "Acueducto y Alcantarillado" → "acueducto-y-alcantarillado"
- "PQRSD 2026" → "pqrsd-2026"
- "Noticias: Nueva Obra" → "noticias-nueva-obra"

### IUploadedFileIIS

Handles file operations on IIS server.

<ParamField name="UploadedFileImage" type="method">
  Uploads file to specified folder, returns filename
</ParamField>

<ParamField name="DeleteConfirmed" type="method">
  Removes file from server storage
</ParamField>

**Storage locations:**
- `blog/` - Blog cover images
- `categories/` - Category images
- `filesBiddingParticipant/` - Proposal PDFs
- `filesDocuments/` - Legal document files

```csharp

```

**Used for:**
- Bidding participant confirmations
- PQRSD submission receipts

## Service Layer Patterns

### Dependency Injection

All services are registered in `Startup.cs` and injected via constructor:

```csharp
services.AddScoped<IBiddingParticipantService, BiddingParticipantService>();
services.AddScoped<IBlogService, BlogService>();
services.AddScoped<ICategoryService, CategoryService>();
services.AddScoped<IDocumentService, DocumentService>();
```

### AutoMapper Usage

DTOs are mapped to entities and vice versa:

```csharp
var dto = _mapper.Map<BiddingParticipantDTO>(entity);
var entity = _mapper.Map<BiddingParticipant>(createDto);
```

### Error Handling

Services with complex operations use transactions:

```csharp
using var transaction = _context.Database.BeginTransaction();
try
{
    // Operations
    transaction.Commit();
}
catch (Exception ex)
{
    transaction.Rollback();
}
```

### Async/Await

All database operations are asynchronous for better performance.
